# L0 v8.3 Superseded Note

**Status:** historical / superseded
**Date:** 2026-05-21
**Superseded by:** L0 v8.4

L0 v8.3 was an internal, pre-provisional agentic-client boundary prototype
derived from a Cline-oriented integration attempt. The Cline-specific path
did not become the current operating direction and is not part of the current
release surface.

## What Is Retired

- Cline as a recommended or current client path.
- `mobius-mmv-cline` as a current release-facing model alias.
- Pseudo-Cline harness results as support for current RC3.3 claims.
- Cline-specific output-contract language in current L0 documents.
- The framing "Cline addon -> 8.3 generalization" as an active project.

## What Survives Into v8.4

The useful client-agnostic ideas survive in L0 v8.4:

- agentic runtime boundary;
- tool readiness before answer commitment;
- grounded vocabulary constraints;
- profile separation between client conventions and Core governance;
- public trace boundary.

These are now described without Cline-specific obligations in
`prompts/l0_integrated_v8_4.md` and
`prompts/mobius_l0_v8_4_protocol.json`.

## Current Reading Rule

When older files mention v8.3, read those passages as historical evidence
unless a current v8.4 document explicitly re-adopts the point.
