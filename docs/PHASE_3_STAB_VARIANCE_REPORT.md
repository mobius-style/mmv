## Notes

### Default mode (N=5, variance 1.81%)

Stable per spec v1.4.1 §7.8.6.1. The σ=0.55 measurement is
consistent with the prior σ=0.65 floor estimate; variance% 1.81% is
well within the 6.5% gate. Phase 3 main flow + stab Commits 41-46
introduce no detectable drift in default-mode behavior.

### Selective primary (N=2, variance 6.96%)

The N=2 sample produced one run at 32 (above gate 31) and one at 29
(within informational-pass band). The variance% of 6.96% slightly
exceeds the 6.5% gate due to the high range (3) being divided by a
small sample. With N=2 this is expected high-variance noise rather
than a true-mean signal. The mean 30.50 is **above** the
informational-pass floor (gate − 2σ = 29.70) and one run reached
the strict gate of 31.

Cumulative selective measurements (Phase 2 final-verify single-run

### Full primary (N=2, variance 2.40%)

## Methodology

Each run is a fresh subprocess of `run_33_scenarios.py` +
`scenario_runner.py`. The subprocess boundary resets the ME5
singleton, providing cold-cache behavior equivalent to a daily
process restart. The cumulative cold-load cost per run is
approximately 5 s; remaining time is qwen3.5:9b synthesis.

Variance % is interpreted relative to a stochastic floor σ ≈ 0.65
(spec v1.4.1 §7.8.6.1). Variance ≈ σ/mean × 100. At mean=31, the
floor σ=0.65 corresponds to ~2.1% variance; the spec gate 6.5%
allows roughly 3σ headroom.

## Sample-size limitation

Phase 3 stab budget allocated N=5 for default mode (Commit 42, the
metric most under question for informational-pass justification)
and N=2 for selective + full (Commit 47, sufficient for variance
trend visibility). Phase 4 will re-evaluate at N=5 across all 3
modes when the cumulative budget allows.
