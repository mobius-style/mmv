# MOBIUS MMV — Embedding Model Standardization Rule

**Status:** Constitutional-level rule (promulgated 2026-04-23, Evolution Log
cycle `cyc_20260423_production_quality_deep_fix`; Box A exception retired
2026-05-21; Box W naming normalized 2026-06-09).
**Authority:** Runtime-enforced where reasonable; human-enforced at
code-review/cycle boundaries for new Box additions or embedding operations.

## Rule

All indexes generated by MOBIUS MMV or its derivatives **MUST** use
`intfloat/multilingual-e5-large` (ME5) as the sentence embedding model.

The rule applies to:

- Every Box that generates or consumes a vector index (Box 0, Box A, Box W,
  Box X, Box M context index, and any future vector Box, including Box B/C if
  those reserved document slots are later vectorized).
- Any ad-hoc embedding operation (for example, a one-off similarity
  computation in a new adapter or tool) — the same model must be used.

## Rationale

1. **Cross-lingual consistency.** MOBIUS serves EN / JA / ZH users from a
   single runtime. A single multilingual encoder (ME5-large, 1024-dim,
   cross-lingual by design) avoids per-language retrieval drift.

2. **Elimination of a systemic failure mode.** Production diagnostic on
   2026-04-23 identified embedding-model drift as the primary root cause of
   Box 0 retrieval mis-hits (ME5-English vs MiniLM-heuristic in Box 0 caused
   `貴方の特徴` → persona-safety chunk instead of MOBIUS identity chunks).
   Standardizing the embedding model at the constitutional level closes the
   class of bugs rather than patching instances.

3. **Retrieval comparability.** With a single encoder family, similarity
   scores across Boxes become numerically comparable, enabling more
   principled cross-Box ranking and adjudication.

4. **Operational simplicity.** One model to cache, load, monitor, and
   upgrade.

## Scope and exceptions

### In scope (MUST comply)

- Box 0 (self-reference). Migrated to ME5 on 2026-04-23.
- Box A (Mobius-foundational material; user docs -> box_1-3 user space).
  Compliant since the 2026-05-21 rebuild.
- Box W (Wikipedia). Uses ME5 since before this rule was promulgated. Older
  code and tests may still expose the legacy selector id `B` for this path.
- Box X (curated external durable knowledge). Must comply; runtime
  confirmation required (see Box Inventory below).
- Box M (session memory and capsule memory indexes). Compliant; capsule
  index migrated to ME5 on 2026-05-21.
- Box B / Box C only if activated as vector document slots in the future.
- Any new vector Box added post-promulgation.

### Not in scope

- External embedding calls inside third-party libraries that we do not
  control (for example, if a downstream tool imports a different sentence
  encoder, we do not forbid the import — we only forbid MOBIUS code from
  depending on a non-ME5 encoder for its own indexes).

## Runtime check

`CustomRagAdapter` (the shared adapter class used by Box 0 and Box A) now
accepts a `model_name` argument. Default value is ME5. Callers that
instantiate it with a different model are responsible for justifying the
deviation in a dated experiment or migration note.

Embedding device selection is controlled by `MMV_EMBEDDING_DEVICE`:

- `auto` (default): use CUDA when available, otherwise CPU.
- `cuda` / `cuda:0`: force CUDA, falling back to CPU if unavailable.
- `cpu`: force CPU for LLM co-residency or deterministic low-resource runs.

Tests that exercise new Box additions should assert on the manifest's
`model` field to lock in the rule.

## Migration guide

For a Box currently on a non-ME5 model:

1. Backup: `cp -r data/<box>/ data/<box>_<oldmodel>_backup_<date>/`.
2. Instantiate the adapter with `model_name="intfloat/multilingual-e5-large"`
   (or via the ME5 default).
3. Delete `<box>/custom_index.faiss`, `<box>/custom_chunks.jsonl`, and
   `<box>/index_manifest.json` to force rebuild.
4. Call `.load()` on the adapter — the rebuild will run automatically
   against the existing corpus directory.
5. Verify the new `index_manifest.json` has
   `"model": "intfloat/multilingual-e5-large"`.
6. Run a retrieval sanity check (language-diverse queries) before committing.

## Revision history

- 2026-04-23: Rule promulgated, Box 0 migrated to ME5. Prior state: Box 0
  used MiniLM-L6-v2, which produced semantic drift on Japanese queries
  (diagnostic: `cyc_20260423_production_quality_deep_fix`).
- 2026-05-21: Box A transitional exception retired. Box A corpus rebuilt
  with ME5-large so MMV vector indexes use one shared embedding model.
- 2026-05-21: Capsule memory FAISS rebuild path migrated from MiniLM
  384-dim to ME5-large 1024-dim.
- 2026-05-21: Runtime embedding device policy changed from CPU-forced to
  `MMV_EMBEDDING_DEVICE=auto`, using CUDA when available.
- 2026-06-09: Normalized Wikipedia naming to Box W. Box B/C are reserved
  document slots, not Wikipedia; legacy selector id `B` remains compatibility
  terminology only.
